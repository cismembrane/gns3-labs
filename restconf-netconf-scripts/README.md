# RESTCONF and NETCONF Scripts

Python scripts that talk to a Cisco IOS XE device over RESTCONF and NETCONF. The labs in this repo configure the same kind of gear from the CLI or with Ansible. These scripts do it through the YANG-modeled APIs instead.

All five scripts target the [DevNet Always-On IOS XE sandbox](https://devnetsandbox.cisco.com/) (`10.10.20.48`, user `developer`, password `C1sco12345`).

## Scripts

| Script                        | What it does                                                                                                   | Writes config |
|-------------------------------|----------------------------------------------------------------------------------------------------------------|---------------|
| `01-restconf-discover.py`     | GETs the RESTCONF API root and the device hostname. Run this first to confirm RESTCONF is reachable.           | No            |
| `02-restconf-interfaces.py`   | GETs interface state through the IETF standard model and the IOS XE native model. Prints both.                 | No            |
| `03-restconf-edit-config.py`  | PUT, GET, PATCH, GET, DELETE, GET on a Loopback. Shows the difference between PUT (replace) and PATCH (merge). | Yes           |
| `04-restconf-static-route.py` | Same sequence on a static route (`192.0.2.0/24 -> Null0`). Dumps the full static route table after the PUT.    | Yes           |
| `05-netconf-static-route.py`  | Same static route sequence but over NETCONF on port 830 using `ncclient`. Raw XML edit-config payloads.        | Yes           |

## Reference

`yang-modules.json` is the `ietf-yang-library:module` list pulled from the sandbox. Kept around for looking up which models the device advertises. Not generated by these scripts.

## Requirements

Python 3.8+ and:

```
requests
urllib3
ncclient
```

Install with:

```
pip install -r requirements.txt
```

## Running

Each script is standalone. No CLI args.

```
python 01-restconf-discover.py
python 02-restconf-interfaces.py
python 03-restconf-edit-config.py
python 04-restconf-static-route.py
python 05-netconf-static-route.py
```

If the sandbox is unreachable or already in use, RESTCONF calls time out at 15 seconds and NETCONF fails to open the SSH session.

## Notes

TLS verification is off (`verify=False`) and `InsecureRequestWarning` is suppressed because the sandbox uses a self-signed cert. NETCONF host key checking is off for the same reason.

Credentials and the target IP are hardcoded at the top of each script. They are the public sandbox values. Change them there if you point the scripts at your own gear.

Scripts 03, 04, and 05 write to running config and clean up after themselves with a DELETE at the end. If a script exits before the DELETE runs, the device may still have the Loopback or route.
